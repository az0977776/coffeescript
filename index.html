CoffeeScript

Introduction

Created by Jeremy Ashkenas in 2009, CoffeeScript was designed as a ``little language that compiles into JavaScript''.

From December of 2009 to December of 2010, the project grew in size, attracting over 300 page hits on the site per day.

The syntax is primarily influenced by Python and Ruby, and the CoffeeScript compiler was initially written in Ruby before being re-written in Coffeescript.

Installation

To Install (with npm): 

	  npm install -g coffee-script
To Install (without npm): 

	  git clone https://github.com/jashkenas/coffeescript.git
	  sudo coffeescript/bin/cake install

Ease of use

Assignment is easy in CoffeeScript; with Python-esque syntax: 

	    number = 42;
	    opposite = true;
Functions are equally simple: 

	    square = (x) -> x * x
	    // This function squares "x"
To call the function, you can either use square(3) or square 3. Parentheses are not necessary unless the function takes no parameters.

Objects

Objects in CoffeeScript can be defined easily: 

	  math =
	    root: Math.sqrt
	    square: square
	    cube: (x) -> x * square x
This code defines an object math, with functions root, square, and cube.

Lists

Lists in CoffeeScript are defined the same way as they are in JavaScript: 

	  list = [1, 2, 3, 4, 5, 6, 7, 8, 9]
However, array splicing can be done using either "..", or "...". Two dots signify that it is inclusive, while three dots signify that the splicing will be exclusive. The first index is always included.

Ex: 

	    list[0..2] // [1, 2, 3]
	    list[0...2] // [1, 2]
Negative indexing is supported. 

	  middle = list[3...-2] // [4, 5, 6, 7]

Aliases

A list of CoffeeScript expressions and its equivalent Javascript: 

	  is // ===
	  isnt // !==
	  not // !
	  and // &&
	  or // ||
	  true, yes, on // true
	  false, no, off // false
	  @, this // this
	  of // in
	  a ** b // Math.pow(a,b)

Boolean Operators

Boolean Operators in CoffeeScript are very literal, and easily understood: 

	  status = good if happy // status = good, if happy == true

	  if happy and glad
	    nap()
	    sleep()
Similarly to Python, indentation is extremely important. However, there are no colons.
